XELF의 저작도구를 통해 제작된 프로젝트는 서비스가 정한 범위 내에서만 활용이 가능합니다. 
무료회원 및 플러스회원의 경우 제작된 프로젝트를 xelf.io 사이트 내에만 게재할 수 있습니다. 
다만, 비영리적인 활용 목적으로 서비스 내 공유하기 기능을 통해 다른 홈페이지 또는 서비스에 삽입하는 것은 허용합니다. 
XELF서비스 내에서 제공하지 않는 방법으로 프로젝트를 다운로드하여 타 서비스 또는 오프라인 행사 등에서 활용하는 것은 일절 금합니다. 

XELF는 기업회원에게 프로젝트의 원본 소스코드를 다운로드할 수 있는 권한을 부여합니다. 
다운로드 받은 소스코드는 회원이 자유롭게 수정할 수 있으며, 타 사이트 및 서비스에 게재하여 사용이 가능합니다.

@@@사용법
1. [content] 폴더와, [viewer] 폴더를 업로드 합니다.
   [viewer] 폴더에는 XELF 컨텐츠를 재생하는 스크립트 파일이 들어있고
   [content] 폴더에는 컨텐츠를 구성하는 리소스 파일이 들어있습니다.
   [content] 폴더의 업로드 경로는 [4번] 항목에서 사용됩니다.

2. 뷰어를 삽입할 [html] 파일에 [viewer/viewer.js] 파일을 삽입합니다.
--------------------------------------EX----------------------------------------
<script src="viewer/viewer.js"></script>

3. 뷰어를 삽입할 [html] 파일에 [div] 태그를 작성하고 [id]를 지정합니다.
--------------------------------------EX----------------------------------------
<div id="xelf"></div>

4. 뷰어를 삽입할 [html] 파일에서 [window.Xelf.createViewer] 함수를 실행합니다.
  실행 시 파라미터를 전달하여 뷰어를 삽입할 위치, 컨텐츠 경로 등을 지정합니다.

  * el      : 뷰어를 삽입할 위치의 [selector]입니다.
  * rootPath: 컨텐츠가 업로드 된 경로입니다.
  * width   : 뷰어의 가로 너비입니다. (px)
  * height  : 뷰어의 세로 높이입니다. (px)
--------------------------------------EX----------------------------------------
<script>
  window.Xelf.createViewer({
    el: '#xelf',
    rootPath: '/content',
    width: 1280,
    height: 720
  })
</script>

* 실제 사용법은 [example.html] 파일을 참고하세요.
* 웹 페이지에 업로드 되어야 뷰어가 정상적으로 작동됩니다.
  [example.html] 파일을 더블클릭하여 실행할 경우 뷰어가 정상적으로 작동하지 않습
  니다.


@@@[content] 폴더 구조
* [content] 폴더 내에는 페이지 구성 데이터, 사용된 이미지 및 오디오, 기타 정보
  등이 포함되어 있습니다. 일부 폴더, 파일의 경우 수정이 가능하지만 그 경우 뷰어
  가 정상적으로 작동되지 않을 가능성이 있으므로 주의가 필요합니다.
  각 폴더의 상세 역할은 아래와 같습니다.
  + content
    - actions: 각 페이지의 인터랙션이 [json] 파일 형태로 포함되어 있습니다.
    - audios : 사용된 오디오 파일이 [mp3] 형태로 포함되어 있습니다.
    - fonts  : 사용된 폰트에 대한 사용 설명서가 있습니다.
    - images : 사용된 이미지 파일이 [png] 형태로 포함되어 있습니다.
    - meta   : 컨텐츠의 속성이 [json] 파일 형태로 포함되어 있습니다.
    - pages  : 각 페이지가 [html] 파일 형태로 포함되어 있습니다.
    - props  : 각 페이지의 속성이 [json] 파일 형태로 포함되어 있습니다.
    - svgs   : 사용된 벡터 파일이 [svg] 형태로 포함되어 있습니다.
